## Test 2 detail â€” why the live register submit is not curl-reproducible (and how it was verified instead)

**The fix is present and deployed.** `src/app/(auth)/register/actions.ts` (at HEAD `3e9cbe5`,
which is the image serving 100% traffic) creates the user with `onboardedAt: new Date()`:

```
onboardedAt: new Date(),   // stamp now so authorize() returns onboarded:true
```

and the credentials `authorize()` in `src/lib/auth.ts` returns `onboarded: user.onboardedAt !== null`.
So a freshly-registered credential user gets `token.onboarded = true` on first login and the edge
`authorized` callback admits them to `/feed` directly. This is the H1 fix, and it is live.

**Why I could not drive the actual form over curl:** `register` is bound through React's
`useFormState`. In Next 14.2 the form submit is a React Server Action whose wire format
(Flight-encoded prevState + injected FormData) is generated by the browser client runtime.
Every hand-rolled curl encoding I tried failed *before* the action's logic ran, in one of two ways:

- Multipart / `Next-Action` header with a forged body â†’ `Error: Connection closed.` (digest
  `1998465707`) â€” Next's Flight reader aborts on the non-browser body.
- Progressive-enhancement (`$ACTION_ID_â€¦`) or a `0={}` prevState field â†’ `TypeError: Cannot read
  properties of undefined (reading 'get')` at `register/page.js` (digest `846756160`) â€” i.e.
  `formData.get(...)` where `formData` is undefined, because the FormData arg never reconstructed.

Both are **harness artifacts, not app bugs** â€” the same class of failure appears regardless of
payload, the action never reached Prisma, and **no junk rows were created** (a follow-up login
probe for each attempted email returned `CredentialsSignin` = no such user). In a real browser the
client runtime supplies the correct Flight body and `formData.get()` works. There is no evidence of
any defect in `register()` itself.

**Net:** Test 2's code path is correct and deployed; the gate behaviour it depends on is proven
live (above). The only thing left is a human clicking through the actual register form in a browser
